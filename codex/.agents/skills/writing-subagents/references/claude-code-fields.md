# Claude Code Agent Frontmatter — Complete Field Reference

> Source: [code.claude.com/docs/en/sub-agents](https://code.claude.com/docs/en/sub-agents)
> Current as of v2.1.81 (2026-03-21)

## Required Fields

| Field | Type | Description |
|---|---|---|
| `name` | string | Unique identifier. Lowercase letters and hyphens only. Becomes the canonical name used for dispatch, @-mentions, and permission rules. |
| `description` | string | Natural language description of when to use this agent. Claude reads this to decide when to delegate. Write as a trigger condition, not a summary of what the agent does. |

## Optional Fields

| Field | Type | Default | Description |
|---|---|---|---|
| `tools` | comma-separated string or array | *(inherits all)* | Allowlist of tools the agent can use. Supports `Agent(type1, type2)` syntax to restrict which subagents can be spawned (only for agents running as main thread via `claude --agent`). |
| `disallowedTools` | comma-separated string or array | *(none)* | Denylist — removed from inherited or specified tools. Applied before `tools` when both are set. A tool in both lists is removed. |
| `model` | string | `inherit` | `sonnet`, `opus`, `haiku`, a full model ID (e.g., `claude-opus-4-6`), or `inherit`. |
| `permissionMode` | string | `default` | `default` — standard prompts. `acceptEdits` — auto-accept file edits. `dontAsk` — auto-deny (allowed tools still work). `bypassPermissions` — skip prompts entirely. `plan` — read-only exploration. |
| `maxTurns` | integer | *(unlimited)* | Maximum agentic turns (tool-call cycles) before the agent stops. |
| `skills` | list of strings | *(none)* | Skill names to preload. Full skill content is injected at startup. Subagents do NOT inherit skills from the parent. |
| `mcpServers` | list | *(inherits)* | MCP servers scoped to this agent. Each entry is a string (referencing a configured server) or an inline object (full MCP config). |
| `hooks` | object | *(none)* | Lifecycle hooks scoped to this agent. Same format as `settings.json` hooks. `Stop` hooks are converted to `SubagentStop` at runtime. Plugin agents cannot use this field. |
| `memory` | string | *(none)* | Persistent cross-session memory: `user` (`~/.claude/agent-memory/<name>/`), `project` (`.claude/agent-memory/<name>/`), or `local` (`.claude/agent-memory-local/<name>/`). First 200 lines of `MEMORY.md` injected at startup. Added v2.1.33. |
| `background` | boolean | `false` | Always run as a background task. Permissions pre-approved at launch; missing permissions auto-denied at runtime. |
| `effort` | string | *(session default)* | Override effort level: `low`, `medium`, `high`, `max` (Opus 4.6 only). Added ~v2.1.78. |
| `isolation` | string | *(none)* | Set to `worktree` to run in a temporary git worktree. Auto-cleaned if no changes. |
| `color` | string | *(auto)* | UI badge color (e.g., `blue`, `purple`, `yellow`). Generated by `/agents` UI. |

## tools / disallowedTools Interaction

| Scenario | Result |
|---|---|
| Both omitted | Agent inherits all tools from parent (including MCP) |
| `tools` only | Strict allowlist — only listed tools available |
| `disallowedTools` only | Inherits everything except listed tools |
| Both set | `disallowedTools` applied first, then `tools` resolved against remainder |

## Agent(type) Syntax

Restricts which subagent types can be spawned when this agent runs as main thread (`claude --agent`):

```yaml
tools: Agent(worker, researcher), Read, Bash
```

- `Agent` (no parens) — allow spawning any subagent
- `Agent(type1, type2)` — only listed types
- Omit `Agent` entirely — cannot spawn subagents

Note: Subagents themselves cannot spawn other subagents regardless of this setting.

## Storage Locations (priority order)

| Priority | Location | Scope |
|---|---|---|
| 1 (highest) | `--agents` CLI flag (JSON) | Current session only |
| 2 | `.claude/agents/` | Current project |
| 3 | `~/.claude/agents/` | User (all projects) |
| 4 (lowest) | Plugin `agents/` directory | Where plugin is enabled |

Same `name` → higher-priority location wins. Agents load at session start; manually added files need a restart or `/agents`.

## Built-in Subagents

| Agent | Model | Tools | Purpose |
|---|---|---|---|
| Explore | Haiku | Read-only | Fast codebase search. Thoroughness: `quick`, `medium`, `very thorough`. |
| Plan | Inherit | Read-only | Research for plan mode. |
| General-purpose | Inherit | All | Complex multi-step tasks. |
| Bash | Inherit | Terminal | Terminal commands in separate context. |

## Plugin Agent Restrictions

Plugin-provided agents cannot use `hooks`, `mcpServers`, or `permissionMode` — these fields are silently ignored for security. Copy the agent to `.claude/agents/` to unlock them.

## Dispatch Mechanisms

- **Automatic**: Claude matches task against agent `description` fields
- **@-mention**: `@"research (agent)"` — guarantees the specific agent runs
- **CLI flag**: `claude --agent research` — main thread runs as that agent
- **Settings**: `"agent": "research"` in `settings.json` for project default
- **Permission rules**: `Agent(my-agent)` in allow/deny lists

## Context Behavior

- Each invocation creates a fresh context (no parent history)
- Only channel from parent to subagent is the Agent tool's prompt string
- Project `CLAUDE.md` is injected via settingSources
- Skills must be listed explicitly in `skills:` — not inherited
- Resume via `SendMessage({to: agentId})` — preserves full history
- Auto-compaction at ~95% capacity
